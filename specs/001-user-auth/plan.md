# Implementation Plan: Authentication & User Management

**Branch**: `001-user-auth` | **Date**: 2026-04-08 | **Spec**: [spec.md](spec.md)

## Summary

Enable parents to register and manage a family account, create player profiles for their kids, and allow kids to select their profile and enter the game without a password. Uses JWT with HttpOnly refresh token cookies for persistent sessions and SendGrid for password reset emails.

## Technical Context

**Language/Version**: .NET 8 (C#) вЂ” backend | TypeScript / React 18 вЂ” frontend
**Primary Dependencies**: ASP.NET Core, Entity Framework Core, MediatR, IPasswordHasher (backend) | React Router, Axios (frontend) | SendGrid (email)
**Storage**: PostgreSQL (EF Core code-first migrations) вЂ” FamilyAccount, PlayerProfile, RefreshToken, PasswordResetToken tables
**Testing**: xUnit (backend) | Jest + React Testing Library (frontend)
**Target Platform**: Web application (browser-based, desktop + tablet)
**Performance Goals**: Registration < 2 min | Login < 30s | Profile select < 5s (per spec SC-001 to SC-003)
**Constraints**: HttpOnly cookies only (no localStorage for tokens), secrets in environment variables only
**Scale/Scope**: Family-scale вЂ” tens of accounts, up to 10 profiles per account
**Architecture**: Full MediatR CQRS вЂ” all controllers dispatch via IMediator, all handlers return unified `Result<T>`, pipeline behaviors for validation/logging/error handling
